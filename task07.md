# Task 07: 重构任务拆解页为"下一步操作指南" + 首页/Footer 修复

## 背景

设计书中不应该包含分步 task。在真实流程中，task 是用户拿着设计书去跟 AI 大模型对话时逐步生成和调整的（就像我们做志远项目一样）。

本站的工作到"输出三份设计书"就基本完成了，之后需要教会用户拿着设计书怎么往下走。

## 目标

1. **重构 tasks.html**：从"任务拆解生成器"改为"下一步操作指南"——手把手教用户从设计书走到开发
2. **修改 design.html**：去掉设计书中的"开发阶段建议/任务拆解"相关提示词内容（如果有的话）
3. 首页标题换行优化
4. Footer 加版本号

第一步：创建项目文件夹
============================

标题："第一步：在电脑上创建项目文件夹"

说明："所有的项目文件都会放在这个文件夹里，包括设计书、代码、资源等。"

项目文件夹名输入
  label："给项目文件夹起个英文名（小写字母、数字和短横线）"
  input 输入框
  placeholder："比如：career-helper、my-app、reading-tool"
  默认值："my-project"
  实时校验：只允许小写字母、数字、短横线
  校验不通过时红色提示

操作步骤（Windows 和 Mac 两栏并排显示，移动端上下堆叠）：

  Windows：
  ```
  1. 打开 PowerShell
  2. 输入以下命令（点击复制）：

  mkdir C:\Users\你的用户名\{文件夹名}
  cd C:\Users\你的用户名\{文件夹名}
  ```
  
  Mac：
  ```
  1. 打开终端（Terminal）
  2. 输入以下命令（点击复制）：
  
  mkdir ~/Projects/{文件夹名}
  cd ~/Projects/{文件夹名}
  ```

  {文件夹名} 实时替换为用户输入的值。
  每条命令旁边有「复制」按钮。

文件夹名保存到 localStorage 项目数据的 folder_name 字段。

第四步：用 AI 大模型生成开发任务
============================

标题："第四步：让 AI 帮你把设计书拆解成开发任务"

说明（青色左边框醒目提示卡）：
  "注意：开发任务不是本站生成的。你需要打开一个 AI 大模型（如 Claude、ChatGPT），把设计书发给它，跟它对话来拆解任务。
   这个过程是交互式的——AI 给你一个初步方案，你看了觉得不对可以让它调整，最终形成你满意的任务清单。"

操作步骤：

  步骤 1：打开你常用的 AI 大模型
  
  AI 平台链接区（横排按钮，新标签页打开）：
    Claude → https://claude.ai
    ChatGPT → https://chatgpt.com
    豆包 → https://www.doubao.com
    Kimi → https://kimi.moonshot.cn
    通义千问 → https://tongyi.aliyun.com
    DeepSeek → https://chat.deepseek.com

  步骤 2：把下面这段话发给 AI（连同你的设计书内容）
  
  按钮（btn-primary）：「生成任务拆解提示词」
  
  提示词展示区（点击按钮后显示）：
    深色代码风格背景框
    右上角「复制」按钮
    
    提示词内容：
    ```
    我是一个零基础的非程序员，想用 AI 编程工具做一个软件项目。
    下面是我的项目设计书，请帮我把它拆解成一步步的开发任务（task）。
    
    要求：
    1. 每个 task 只做一件小事，宁可多拆几个也不要合并太多
    2. 前几个 task 尽量简单，后面逐步增加复杂度
    3. task01 应该是项目初始化（创建基础文件结构）
    4. 每个 task 写清楚：编号、标题、目标、具体步骤、完成标志
    5. 最后附上一份 task01 的完整执行指令，我可以直接复制给 AI 编程工具
    
    项目设计书：
    ---
    {design_doc_content}
    ---
    
    请开始拆解。如果有不清楚的地方，先问我再拆。
    ```

    {design_doc_content} 从 localStorage 读取 design_doc，优先 ai_coding 版，没有就用 standard 版，再没有用 popular 版。
    如果都为空，提示"请先生成设计书"。

  步骤 3：和 AI 来回讨论，直到你满意
  
  提示文字：
    "AI 给出任务清单后，你可以：
     • 觉得某个 task 太复杂 → 让它拆细
     • 觉得缺了什么功能 → 让它补上
     • 觉得顺序不对 → 让它调整
     • 满意后 → 继续下一步"

============================
第五步：开始逐个执行任务
============================

标题："第五步：用 AI 编程工具逐个执行任务"

说明："现在你有了任务清单，在 AI 编程工具中一个一个执行就行。"

操作流程卡片（醒目样式，步骤式展示）：

  ① 打开你的 AI 编程工具，进入项目文件夹
  
  ② 把 task01 的内容发给它（或者直接说"请读取设计书，执行 task01"）
  
  ③ AI 执行完毕后，自己检查一下结果
  
  ④ 没问题 → 继续发 task02
     有问题 → 告诉 AI 哪里不对，让它修
  
  ⑤ 重复 ②-④ 直到全部任务完成
  
  ⑥ 产品上线 🚀

提示卡片（青色左边框）：
  "小技巧：
   • 如果用 Claude Code，可以直接说「请读取 CLAUDE.md，然后执行 task01」
   • 每完成一个 task，建议用 Git 提交一次代码（AI 工具通常会帮你做）
   • 遇到反复失败的问题，可以回到 AI 大模型对话中讨论，调整方案后再执行"
