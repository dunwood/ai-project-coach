  --- 基础知识 ---

  1. "什么是命令行（CLI）？"
     ```
     命令行（CLI，Command Line Interface）就是一个纯文字的操作界面——没有按钮可以点，你通过输入文字命令来控制电脑。

     在不同系统上它叫不同的名字：
     - Windows：PowerShell（推荐）或 CMD（命令提示符）
     - Mac：终端（Terminal）
     - Linux：终端（Terminal）

     怎么打开：
     - Windows：按 Win 键，输入"PowerShell"，回车
     - Mac：按 Cmd+空格，输入"Terminal"，回车

     你只需要掌握几个最基础的命令：
     - cd 文件夹名：进入某个文件夹（比如 cd Desktop 就是进入桌面）
     - cd ..：退回上一层文件夹
     - dir（Windows）或 ls（Mac/Linux）：查看当前文件夹里有什么文件
     - 上下箭头键：翻看之前输入过的命令

     为什么需要它：
     Claude Code 等 AI 编程工具运行在命令行里。你不需要精通命令行，只需要会打开它、进入项目文件夹、输入一条启动命令就够了。剩下的交给 AI。

     举个例子，整个操作可能就是这三步：
     1. 打开 PowerShell
     2. 输入：cd C:\Users\你的用户名\你的项目文件夹
     3. 输入：claude
     然后你就进入了 Claude Code，可以开始跟 AI 对话了。
     ```

  2. "什么是 VS Code？需要装吗？"
     ```
     VS Code（Visual Studio Code）是一个免费的代码编辑器，由微软出品。你可以把它理解为一个"高级记事本"——专门用来查看和编辑代码文件的。

     下载地址：https://code.visualstudio.com

     它能帮你做什么：
     - 打开整个项目文件夹，一目了然地看到所有文件
     - 代码文件会有语法高亮（不同类型的内容用不同颜色显示），方便阅读
     - 内置终端（命令行），不用另外打开 PowerShell
     - 可以直接在里面编辑文件，比如改个文字、调个颜色

     需要装吗：
     - 如果你用 Cursor 或 TRAE：不需要装 VS Code，它们本身就是类似的编辑器
     - 如果你用 Claude Code（命令行工具）：强烈建议装一个 VS Code，用来查看项目文件和检查 AI 生成的代码
     - 如果你什么都还没装：先装 VS Code，它是最基础的工具

     安装后的第一件事：
     1. 打开 VS Code
     2. 点击左上角"文件" → "打开文件夹"
     3. 选择你的项目文件夹
     4. 左侧就能看到所有文件了
     ```

  --- AI 编程工具 ---

  3. "如何用 Claude Code 执行任务"
     ```
     Claude Code 是目前最强的 AI 编程工具之一，在命令行中运行。它能自主读写文件、运行代码、调试错误，像一个真正的程序员搭档。

     前置条件：
     - 需要 Claude Pro 或 Max 订阅（claude.ai 付费）
     - 需要安装 Node.js（下载地址：https://nodejs.org，选 LTS 版本）

     安装步骤：
     1. 安装 Node.js 后，打开 PowerShell（Windows）或终端（Mac）
     2. 输入：npm install -g @anthropic-ai/claude-code
     3. 等待安装完成

     开始使用：
     1. 在 PowerShell 中进入你的项目文件夹：cd C:\Users\你的用户名\你的项目文件夹
     2. 输入：claude
     3. 首次使用会弹出浏览器让你登录 Claude 账号，登录即可
     4. 登录成功后，你就进入了 Claude Code 的对话界面
     5. 把上面生成的「首轮开发 Prompt」粘贴进去，回车
     6. Claude Code 会自动创建文件、写代码、调试
     7. 完成后检查结果，有问题直接告诉它，它会修复

     高效技巧：
     - 启动时加 --dangerously-skip-permissions 可以跳过反复确认，全自动执行
     - 每完成一个 task，直接说"请读取 task02.md 执行任务"即可继续下一个
     - 遇到问题可以截图或复制错误信息给它
     ```

## 任务拆解提示词模板

```
请根据以下设计书内容，帮我把项目拆解成一步步的开发任务。

要求：
1. 按阶段分组：
   - 阶段一：项目骨架搭建（创建文件结构、基础页面、全局样式）
   - 阶段二：核心功能开发（实现主要功能模块）
   - 阶段三：打磨优化（UI美化、移动端适配、性能优化、错误处理）
   - 阶段四：部署上线（部署到服务器、域名配置、最终测试）

2. 每个任务包含：
   - 任务编号（task01, task02, task03...）
   - 任务标题
   - 目标（一句话说清楚做什么）
   - 具体步骤（列出要做的事）
   - 完成标志（怎样算做完了）

3. 每个任务只做一件事，宁可多拆几个任务也不要合并太多内容
4. 前几个任务尽量简单，后面逐步增加复杂度
5. 第一个任务（task01）应该是项目初始化：创建文件夹、基础文件、初始化 Git

设计书内容：
---
{design_doc_content}
---
```

### 变量说明

`{design_doc_content}` 从 localStorage 中读取该项目的 design_doc。优先使用 ai_coding 版本，没有就用 standard 版本，再没有就用 popular 版本。三个版本都为空则提示"请先生成设计书"。
