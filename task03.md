# Task 03: 需求澄清页（核心引擎）

## 目标

实现 clarify.html——整个产品最核心的页面。用户在这里把模糊想法变成清晰的项目定义。

本站不内嵌 AI 对话，而是：
1. 展示用户的原始想法
2. 提供结构化的引导问题，用户可以自己思考填写
3. 一键生成「需求澄清提示词」，用户复制到站外大模型（Claude、ChatGPT、豆包等）进行对话
4. 用户在站外和 AI 对话后，回来把结果填入各字段
5. 所有字段保存到 localStorage

  "和 AI 聊完之后，把结果填入下面的表单："

  14. MVP 范围（第一版做到什么程度）

    副按钮（btn-secondary）：「保存草稿，稍后继续」

      - 弹出 toast 提示"已保存"
      - 不跳转

Footer（nav.js）

## 提示词模板

「生成需求澄清提示词」按钮点击后，根据用户的项目名称和原始想法，生成以下提示词：

```
我想做一个软件项目，请你扮演一个专业的项目经理，通过提问帮我把想法理清楚。

我的想法是：
"""
{用户填写的原始想法}
"""

请你一步步问我以下问题（每次只问 1-2 个，等我回答后再问下一个）：

1. 这个软件核心要解决什么问题？
2. 目标用户是谁？他们在什么场景下会使用？
3. 用户需要输入什么信息？系统给出什么结果？
4. 第一版必须有哪些功能？哪些可以以后再做？
5. 打算做成网页、小程序还是 APP？
6. 需不需要用户登录？需不需要付费功能？需不需要后台管理？
7. 数据从哪里来？
8. 第一版的范围是什么？做到什么程度算完成？

请用大白话跟我对话，不要用技术术语。每次我回答后，帮我总结一下当前已经确定的内容，然后继续问下一个问题。

当所有问题都问完后，请帮我整理一份结构化的项目定义，包含以下字段：
- 项目名称
- 核心目标
- 目标用户
- 使用场景
- 输入内容
- 输出结果
- 核心功能
- 非核心功能
- 平台形态
- 是否需要登录
- 是否需要支付
- 是否需要后台
- 数据来源
- MVP 范围
```

注意：`{用户填写的原始想法}` 替换为该项目 localStorage 中存储的 idea 字段真实内容。

## 数据存储

澄清表单的数据保存在项目的 `clarify_answers` 字段中：

```json
{
  "clarify_answers": {
    "project_name": "高考择业助手",
    "core_goal": "帮高考生和家长选择适合的专业方向",
    "target_users": "高考生及其家长",
    "use_cases": "高考志愿填报前",
    "input_data": "成绩、科目偏好、兴趣爱好、家庭情况",
    "output_result": "专业推荐排名 + 个性化分析报告",
    "core_features": "问卷评测、专业匹配、AI报告生成",
    "non_core_features": "分享功能、历史记录",
    "platform_type": "网页",
    "need_login": "不需要",
    "need_payment": "不需要",
    "need_admin": "不需要",
    "data_source": "自建专业数据库 + AI 实时生成",
    "mvp_scope": "50个热门专业，只做网页版",
    "tech_difficulty": ""
  }
}
```

需要在 js/project.js 的 updateProject 函数支持更新 clarify_answers 字段。

---

## 页面加载逻辑

1. 从 URL 读取 `?id=xxx` 参数
2. 调用 getProject(id) 加载项目数据
3. 如果项目不存在，显示错误提示 + 返回工作台链接
4. 如果项目已有 clarify_answers，预填到表单各字段中（编辑模式）
5. 实时更新进度条
